# Reader

Бессерверный markdown-редактор, где документ целиком живёт внутри ссылки. Ни бэкенда, ни базы, ни аккаунта — URL *и есть* документ.

Демо: <https://olluorg.github.io/reader/>

[English version](./README.md)

## Как это работает

Markdown сжимается (Brotli, где доступен, иначе deflate-raw), опционально шифруется AES-256-GCM и упаковывается во фрагмент URL (часть после `#`). Браузер не отправляет фрагмент в HTTP-запросах, поэтому документ никогда не уходит ни на один сервер — включая тот, что раздаёт само приложение.

- **Пишите.** Markdown рендерится вживую по мере набора.
- **Делитесь.** Нажмите *Share* — получите самодостаточный URL.
- **Шифруйте.** Добавьте пароль, и ссылка станет AES-256-шифротекстом.
- **Роли.** Генерируйте отдельные ссылки для *просмотра*, *комментирования* или *редактирования*.
- **Разбивайте.** Документы, не влезающие в один URL, режутся на связанные части.
- **Версии.** Каждое расшаривание дописывает сжатый диф, так что историю ссылок можно проверить.

## Стек

- [Vite](https://vitejs.dev/) + TypeScript
- [Milkdown](https://milkdown.dev/) (WYSIWYG на ProseMirror)
- Web Crypto API для шифрования
- `CompressionStream` для Brotli / deflate-raw
- IndexedDB для локальной библиотеки (история, закладки, ваши шары)

Хостится на GitHub Pages как полностью статический бандл.

## Локальная разработка

Нужен [Bun](https://bun.sh/).

```sh
bun install
bun run dev      # vite dev-сервер
bun run build    # type-check + продакшен-сборка в dist/
bun run preview  # запустить продакшен-сборку
```

## Деплой

Пуш в `main` запускает `.github/workflows/deploy.yml`, который собирает проект через Bun и публикует `dist/` на GitHub Pages.

## Приватность

Содержимое документа живёт во фрагменте URL (после `#`), а браузер не отправляет фрагмент в HTTP-запросах. Локальная библиотека (история, закладки, ваши шары) хранится в IndexedDB на вашем устройстве. Ничего не синхронизируется и не передаётся наружу.

Сам статический сайт раздаёт GitHub Pages, так что GitHub видит обычные метаданные запроса (IP, user agent, путь) — но не фрагмент, а значит и не ваш документ.

## Ограничения

- URL ограничен ~50 KB, чтобы укладываться в лимиты браузеров и мессенджеров. Длинные документы на этапе шаринга режутся на несколько связанных частей.
- WYSIWYG-редактор отключается на документах больше ~300k символов и переключается в raw-textarea (лимит можно вручную обойти, если хочется попробовать).

## Лицензия

Пока не указана. Без лицензии действует дефолтный копирайт — исходники открыты для чтения, но переиспользование требует разрешения владельца.
